Git/GitHub Project - Julian Fry - May 2019

Using a Version Control System such as Git or GitHub can provide many benefits to developers. A VCS can make collaboration between team members easier and more efficient. They can also track changes that you've made to the program, and store earlier versions of the code. Also, they can act as a backup for your code, just in case any disaster befalls your code.

While Git and GitHub have similar names, they are not the same thing. Git is the actual version control system itself, which is installed on a local machine and manages the history of source code, which is stored in repositories. GitHub is an online service for hosting Git repos, and its purpose is to make collaboration on projects easier, and to provide a backup for the code.

  • Initialize a Git repository in the desired file structure with "git init"
  • Check the status of your Git repository with "git status"
  • If you want your Git repository to ignore files, use the command "touch .gitignore" to add a file that will track folders or files that you want Git to ignore.
  • In order to edit the .gitignore file, use any text editor like gedit, and add the folders orfiles that you would like to track.
  • To add files to the Git staging area, use the command "git add (Insert File)". If you want to add every file in the repo, use the command "git add -A"
  • To remove files from the staging area, use the command "git reset".
